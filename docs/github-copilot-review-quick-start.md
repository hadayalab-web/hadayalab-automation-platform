# GitHub Copilotレビュー - クイックスタートガイド

## 🚀 レビュー開始方法

### 1. GitHub Copilot Chatを開く
- VS Code / Cursorで `Ctrl+L` (Mac: `Cmd+L`) を押す
- または、GitHub Copilot Chatパネルを開く

### 2. レビュー依頼コマンド

以下のコマンドをGitHub Copilot Chatに入力：

```
@docs/github-copilot-review-request.md このファイルをレビューしてください。

特に以下の点を確認してください：
1. n8nワークフロー設計の妥当性
2. JSON形式の正確性（n8nでインポート可能か）
3. 式（expressions）の記述が正しいか
4. エラーハンドリングが適切か
5. セキュリティ設定が適切か

改善提案もお願いします。
```

### 3. 個別ファイルレビュー

#### n8nワークフロー設計のレビュー
```
@n8n-workflows-design.md このワークフロー設計をレビューしてください。

確認ポイント：
- ワークフロー構造が論理的か
- Wait Nodeの使用が適切か（6時間、12時間の待機）
- Switch Nodeの分岐が効率的か
- エラーハンドリングが考慮されているか
```

#### ワークフローJSONのレビュー
```
@workflow-1-trial-onboarding.json このn8nワークフローJSONをレビューしてください。

確認ポイント：
- JSON形式が正しいか（n8nでインポート可能か）
- 式の記述が正しいか（例: {{ $json.body.user_email }}）
- HTML Emailのエスケープが適切か
- 認証情報の参照が適切か
```

#### 実装ガイドのレビュー
```
@README-n8n-implementation.md この実装ガイドをレビューしてください。

確認ポイント：
- セットアップ手順が明確か
- 必要な認証情報がすべて記載されているか
- トラブルシューティングが十分か
- セキュリティ注意事項が記載されているか
```

## 📝 レビュー結果の記録

GitHub Copilotからのレビュー結果は、以下の方法で記録できます：

### 方法1: コメントとして記録
1. GitHub Copilot Chatの回答をコピー
2. `docs/github-copilot-review-feedback.md` に保存

### 方法2: Issueとして記録
1. GitHubリポジトリでIssueを作成
2. レビュー結果をIssueに記載
3. 改善項目をタスクとして整理

## 💡 改善提案のリクエスト例

### パフォーマンス最適化
```
Wait Nodeの6時間、12時間の長時間待機について、より効率的な実装方法はありますか？
n8n Cloudでの実行時間制限を考慮した代替案を提案してください。
```

### セキュリティ強化
```
API Keyの管理方法について、環境変数やn8n Credentialsの使用を推奨しますが、
具体的な実装方法を教えてください。
```

### エラーハンドリング
```
エラーハンドリングノード（Try-Catch）の追加方法と、
エラー時の通知方法を提案してください。
```

# LLMモデル選定ガイド

このプロジェクト（n8nワークフロー自動化プラットフォーム）における最適なLLMモデルの選定ガイドです。

## 🎯 プロジェクト用途別推奨

### このプロジェクトで行う主な作業

1. **n8nワークフロー開発** - JSON構造、ノード設定
2. **ドキュメント作成** - Markdown、README
3. **コードレビュー** - 品質チェック、ベストプラクティス
4. **プロジェクト分析** - アーキテクチャ、改善点
5. **自動化設計** - ワークフロー設計、統合

---

## 🎯 実践的な使い分け

### シナリオ1: n8nワークフロー作成

**推奨: Claude Sonnet 4.5（Cursor）**
```
「Slack通知ワークフローを作成して」
→ Claudeがn8n構造を理解して高品質なJSONを生成
```

### シナリオ2: 既存ワークフローのレビュー

**推奨: GPT-4o（GitHub Copilot）**
```
GitHub Copilot: 「このワークフローをレビューして」
→ セキュリティ、ベストプラクティスをチェック
```

### シナリオ3: プロジェクト全体の分析

**推奨: GPT-4o（GitHub Copilot Workspace）**
```
GitHub Copilot: 「リポジトリ全体を分析して改善点を提案して」
→ リポジトリ全体を理解して包括的な分析
```

### シナリオ4: 複雑なアーキテクチャ設計

**推奨: Claude Opus 4.5**
```
「n8nワークフローと他のシステムを統合する設計を提案して」
→ 深い推論で包括的な設計を生成
```

## 🔄 モデルの切り替えタイミング

### Cursor内でモデルを切り替える場合:

1. Cursor Settings → Models
2. 用途に応じて選択:
   - **通常:** Claude Sonnet 4.5
   - **深い分析:** Claude Opus 4.5
   - **高速試作:** Claude Composer 1

# GitHub Copilot タスク一覧

このドキュメントは、GitHub Copilotに任せられる具体的なタスクとそのプロンプトをまとめています。

## 🎯 推奨タスク順序

### 1. プロジェクト全体分析（最優先）

**GitHub.com → Copilot Chat で実行:**

```
@copilot このリポジトリ全体を分析して、以下の観点から改善点を特定して:
- アーキテクチャの改善点
- コード品質（n8nワークフローJSONの構造）
- ドキュメントの完全性
- テストカバレッジの不足
- セキュリティ懸念（API key管理など）
- パフォーマンス最適化の機会
- ベストプラクティスの適用漏れ

分析結果をIssueとして記録し、優先度（High/Medium/Low）を付けてください。
```

---

### 2. 既存ワークフローのコードレビュー

**GitHub.com → リポジトリ → workflows/manual-hello-world-test.json → Copilot Chat:**

```
@copilot このn8nワークフローをレビューして、以下の観点から改善点を提案して:
- エラーハンドリングの実装
- ノードの最適化
- 命名規則の遵守（workflow-conventions.mdに基づく）
- セキュリティ（機密情報の管理）
- 可読性とメンテナンス性
- ドキュメント化の必要性

改善提案をIssueとして作成してください。
```

---

### 3. テスト戦略の提案

**GitHub.com → Copilot Chat:**

```
@copilot このプロジェクトのテスト戦略を提案して:
- n8nワークフローのテスト方法
- テストワークフローの設計
- 統合テストの計画
- CI/CDパイプラインの改善

具体的なテストワークフローの実装計画をIssueに記録してください。
```

---

### 4. ドキュメントの改善提案

**GitHub.com → Copilot Chat:**

```
@copilot このリポジトリのドキュメントをレビューして:
- 不足しているドキュメント
- 古くなった情報の特定
- ドキュメントの構造改善
- READMEの充実
- APIドキュメントの必要性

改善提案をIssueに記録してください。
```

---

### 5. ワークフローテンプレートの作成

**GitHub.com → Copilot Chat:**

```
@copilot このプロジェクト用のn8nワークフローテンプレートを作成して:
- 一般的なパターン（Webhook、スケジュール、メールトリガー）
- エラーハンドリングを含む標準テンプレート
- ベストプラクティスに準拠した構造

テンプレートファイルを`workflows/templates/`ディレクトリに作成する計画をIssueに記録してください。
```

---

### 6. CI/CDパイプラインの改善

**GitHub.com → .github/workflows/validate-workflows.yml → Copilot Chat:**

```
@copilot このGitHub Actionsワークフローをレビューして:
- 改善点の特定
- 追加すべき検証（Lint、セキュリティスキャンなど）
- パフォーマンス最適化
- 通知の追加

改善提案をIssueに記録してください。
```

---

### 7. セキュリティ監査

**GitHub.com → Copilot Chat:**

```
@copilot このリポジトリのセキュリティ監査を実施して:
- 機密情報の漏洩リスク
- API key管理のベストプラクティス
- 依存関係の脆弱性
- n8nワークフロー内のセキュリティリスク

発見された問題を優先度付きでIssueに記録してください。
```

---

### 8. ワークフロー最適化

**GitHub.com → Copilot Chat:**

```
@copilot 既存のn8nワークフローを最適化する提案をして:
- ノード数の削減
- 処理速度の改善
- コスト最適化
- 再利用性の向上

最適化計画をIssueに記録してください。
```

## 📝 Issue作成用テンプレート

GitHub CopilotがIssueを作成する際に使用するテンプレート：

```markdown
## 概要
[問題や改善点の概要]

## 優先度
- [ ] High
- [ ] Medium
- [ ] Low

## カテゴリ
- [ ] バグ修正
- [ ] 機能追加
- [ ] リファクタリング
- [ ] ドキュメント
- [ ] テスト
- [ ] パフォーマンス
- [ ] セキュリティ

## 詳細
[詳細な説明]

## 提案する解決策
[解決策の提案]

## 関連ファイル
- `path/to/file`

## 見積もり
[時間見積もり]
```

## 🔄 定期的なタスク

### 週次タスク

```
@copilot このリポジトリの変更履歴を分析して、今週の改善点を提案して:
- 新規追加されたワークフローのレビュー
- ドキュメントの更新が必要な箇所
- 技術的負債の蓄積

週次レビューIssueを作成してください。
```

### 月次タスク

```
@copilot このリポジトリの月次レポートを作成して:
- 追加された機能
- 解決されたIssue
- 残っている技術的負債
- 次月の優先度

月次レポートIssueを作成してください。
```

## 🎯 効果的な使い方

### 1. 明確な指示を出す

❌ 悪い例: "レビューして"
✅ 良い例: "セキュリティとパフォーマンスの観点からレビューして、Issueに記録して"

### 2. コンテキストを提供する

関連ドキュメントやファイルを参照させる:
```
@copilot workflow-conventions.mdとdocs/hadayalab-automation-platform-SSOT.mdを参照して、
このワークフローが規約に準拠しているか確認して
```

### 3. 段階的に依頼する

大きなタスクは分割:
```
1. まず全体を分析
2. 優先度の高いIssueを作成
3. 具体的な実装計画を作成
```

## 🔧 Section 3: 設定手順

### 3.1 Cursor Settingsでの設定

```yaml
ステップ1: 設定を開く
  1. Cursor Settings → Models
  2. 現在ONになっているモデルを確認

ステップ2: 最強組み合わせを適用

最強構成（Extra High + Composer 1）を選択する場合:
  1. ✅ GPT-5.2 Extra High: ON
  2. ✅ GPT-5.2 Extra High Fast: ON
  3. ✅ Claude Composer 1: ON（パフォーマンス最適化のため必須）
  4. ❌ その他すべて: OFF
     - GPT-5.2 High（Extra Highがある場合は不要）
     - GPT-5.2 High Fast（Extra High Fastがある場合は不要）
     - GPT-5.1系すべて
     - Claude Sonnet 4.5（Composer 1で十分）
     - Claude Opus 4.5（Composer 1で十分）
     - Gemini 3 Pro
     - その他すべて

コスト効率重視（High）を選択する場合:
  1. ✅ GPT-5.2 High: ON
  2. ✅ GPT-5.2 High Fast: ON
  3. ❌ その他すべて: OFF

ステップ3: 動作確認
  1. 簡単なコード生成タスクで確認
  2. 公式ドキュメント参照タスクで確認
  3. エラー修正タスクで確認
```

### 3.2 モデル選択の仕組み（推測）

```yaml
Cursorのモデル選択フロー:
  1. クレジットベースのフォールバック
     - 優先順位（能力順）:
       1. GPT-5.2 Extra High Fast（最強、高コスト）
       2. GPT-5.2 Extra High（最高品質、高コスト）
       3. GPT-5.2 High Fast（高品質+高速、中コスト）
       4. GPT-5.2 High（高品質、中コスト）
       ↓ クレジット不足でフォールバック
       5. Claude Sonnet 4.5（高品質、低コスト）

  2. 用途別使い分けは不可
     - ⚠️ Cursorでは用途に応じた使い分けができない
     - ✅ コード生成・開発タスクに最適化した組み合わせのみを推奨

  3. 推奨設定
     - クレジット十分: GPT-5.2 Extra High系
     - クレジット制限: GPT-5.2 High系
     - Claude系はフォールバックとしても不要（GPT-5.2 High系で十分）
```
